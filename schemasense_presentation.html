01Problem

Engineering diagrams need evidence, not just answers.

  • Diagrams mix symbols, labels, lines and domain conventions.
  • A full-image VLM answer can be correct but hard to audit.
  • SchemaSense exposes intermediate reasoning artifacts.

Design target: keep the system local, understandable and demo-ready.

Black-box answerAudited answer
02Architecture

One local model, multiple logical agents.

  • Qwen3-VL runs through a single local llama.cpp server.
  • Agents are Python roles with specialized prompts.
  • No cloud APIs, LangChain, LangGraph, Streamlit, or React.
Shared inferenceSpecialized prompts
03Agent Roles

Small roles keep the system inspectable.

  • Cartographer — manages spatial coverage.
  • Spotter — turns pixels into symbol evidence.
  • Connector — converts local relationships into graph edges.
  • Reasoner — answers from graph context first.
Role mapI/O contracts
04Local Deployment

The demo stays on one machine.

  • Browser talks to FastAPI over localhost.
  • FastAPI calls pure Python orchestration.
  • llama.cpp serves the Qwen3-VL model.
  • Artifacts are written under organized output folders.
One machineNo cloud
05Pipeline

From pixels to an answer with provenance.

  • Tile the diagram.
  • Detect symbols and boxes.
  • Build a component graph.
  • Answer with graph + visual verification.
End-to-endCached & verified
06Cache & Orchestration

Expensive VLM calls are reused when possible.

  • Image-derived artifacts are stable across questions.
  • The orchestrator checks cache before repeating work.
  • Evaluation can reuse the same visual verification call.
Decision loopPersistent artifacts
07Cartographer

Tiling preserves small details.

  • Large diagrams are split into overlapping tiles.
  • Every detection is projected back to full-image coordinates.
  • Overlays stay aligned with the original image.
TileProject back
08Symbol Spotter

Visual entities become structured detections.

  • Qwen3-VL returns symbols, labels, boxes and confidence.
  • Post-processing normalizes and deduplicates boxes.
  • Detections are visual evidence, not hidden state.
PixelsStructured JSON
09Connector

Detections become a component graph.

  • Each symbol is a node.
  • Local crops ask: what connects directly?
  • Edges store a coarse connection type.
Boxes & cropsNode–edge graph
10Reasoner

Graph-first, then visual verification.

  • The graph provides structure and explainability.
  • The verifier protects against sparse or missing graph evidence.
  • The response records answer provenance.
DecisionProvenance
11Demo Workflow

  • Analyze — upload one image and inspect the answer.

  • Comparison — run baseline vs SchemaSense end-to-end.

13Limitations

The honest story is still strong.

  • Graph extraction can miss small or dense details.
  • Counting remains the hardest recorded category.
  • The hybrid verifier is a practical reliability guard.
  • Future work: better graph validation and symbol schemas.
RisksMitigations
14Why It Matters

Baseline-level answers, with inspectable evidence.

  • Baseline gives strong direct answers.
  • SchemaSense adds detections, graph, cache and provenance.
  • The hybrid verifier prevents graph-only failure cascades.
BaselineSchemaSense
15Closing

SchemaSense is a local, explainable VLM pipeline.

Same model. Same answers. New evidence.

  • Local inference with Qwen3-VL and llama.cpp.
  • Explicit pipeline artifacts for defense and debugging.
  • Hybrid final answering for practical robustness.
